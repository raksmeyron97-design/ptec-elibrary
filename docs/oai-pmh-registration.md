# ឯកសារណែនាំពីការ Validate និងចុះឈ្មោះ OAI-PMH

ឯកសារនេះសង្ខេបពីជំហានដែលយើងបានធ្វើដើម្បី Validate (ផ្ទៀងផ្ទាត់) ប្រព័ន្ធ OAI-PMH របស់បណ្ណាល័យឌីជីថល PTEC និងដំណើរការនៃការចុះឈ្មោះបណ្ណាល័យនេះទៅកាន់ប្រព័ន្ធអន្តរជាតិធំៗ (Aggregators)។

## ១. ការផ្ទៀងផ្ទាត់ប្រព័ន្ធ OAI-PMH (Validation)

យើងបានប្រើប្រាស់ប្រព័ន្ធ **BASE OVAL Validator** (`https://oval.base-search.net/`) ដើម្បីឆែកមើលថាតើតំណភ្ជាប់ (`https://library.ptec.edu.kh/api/oai`) ត្រឹមត្រូវតាមស្តង់ដារដែរឬទេ។

ក្នុងអំឡុងពេលឆែក យើងបានជួបប្រទះ និងដោះស្រាយកំហុស (Errors) ចំនួន ២៖

### កំហុសទី១៖ XML Validation (`dc:title` element declaration)
*   **ស្ថានភាព៖** កំហុសក្លែងក្លាយ (False Positive)
*   **ការពន្យល់៖** OVAL បានរាយការណ៍ថាវាមិនស្គាល់ QName `{http://purl.org/dc/elements/1.1/}title` ទេ។
*   **ដំណោះស្រាយ៖** នេះគឺជាបញ្ហាដែលគេដឹងជាទូទៅចំពោះ OVAL និងកម្មវិធីអាន XML ចាស់ៗ ដែលមិនអាចទាញយកទិន្នន័យពី `http://purl.org` បានដោយសារវេបសាយនោះប្តូរទៅ `https://` (HTTP redirects)។ ទិន្នន័យ OAI-PMH របស់ PTEC គឺគោរពតាមស្តង់ដារ ១០០% ដូច្នេះយើងអាច **រំលងកំហុសនេះបានដោយសុវត្ថិភាព**។

### កំហុសទី២៖ Incremental Harvesting (full granularity)
*   **ស្ថានភាព៖** បានជួសជុលរួចរាល់ (Fixed)
*   **ការពន្យល់៖** ប្រព័ន្ធមិនអាចទាញយកទិន្នន័យបានទេនៅពេលដែលគេ Query ចន្លោះកាលបរិច្ឆេទ (`from` និង `until`)។
*   **ដំណោះស្រាយ៖** យើងបានរកឃើញបញ្ហានៅក្នុងកូដ `lib/oai/records.ts` ដោយសារវាបានរំលងឯកសារទាំងឡាយណាដែលមិនមានថ្ងៃខែកែប្រែ (`updated_at` គឺ `NULL`)។ កូដនេះត្រូវបានកែតម្រូវដោយប្រាប់ប្រព័ន្ធឱ្យទាញយកកាលបរិច្ឆេទបង្កើត (`created_at`) មកជំនួសវិញក្នុងករណីដែល `updated_at` ជា `NULL` ដោយប្រើមុខងារ `.or()` របស់ Supabase។
*   **ជំហានបន្ទាប់៖** កូដដែលបានកែនេះ ត្រូវតែបញ្ជូន (Deploy) ទៅកាន់ Server របស់ Vercel ជាមុនសិន។ បន្ទាប់ពី Deploy រួចរាល់ ការចុច Validate ម្តងទៀតនៅក្នុងប្រព័ន្ធ OVAL នឹងទទួលបានលទ្ធផល **SUCCESS**។

## ២. ដំណើរការនៃការចុះឈ្មោះបណ្ណាល័យ (Aggregator Registration)

ដើម្បីឱ្យឯកសារក្នុងបណ្ណាល័យ PTEC អាចស្វែងរកឃើញជាសកល យើងត្រូវយកតំណ OAI-PMH ទៅចុះឈ្មោះជាមួយប្រព័ន្ធអន្តរជាតិ។ ការចុះឈ្មោះត្រូវធ្វើតាមលំដាប់លំដោយដូចខាងក្រោម៖

### ជំហានទី១៖ OpenDOAR (Directory of Open Access Repositories)
OpenDOAR គឺជាបញ្ជីឈ្មោះគោល។ ប្រព័ន្ធផ្សេងៗទៀត (ដូចជា OpenAIRE) គឺតែងតែទាញយកឈ្មោះបណ្ណាល័យចេញពី OpenDOAR នេះឯង។
*   **តំណភ្ជាប់៖** `https://v2.sherpa.ac.uk/opendoar/`
*   **សកម្មភាពដែលបានធ្វើ៖** បានបំពេញទម្រង់ "Suggest a Repository" សម្រាប់ស្ថាប័ន PTEC (រើសយកប្រភេទ Institutional, មុខវិជ្ជា Social sciences & Science, និងគោលការណ៍ Open access)។
*   **ស្ថានភាព៖** កំពុងរង់ចាំការត្រួតពិនិត្យពីក្រុមការងារ OpenDOAR (អាចចំណាយពេល ១ ទៅ ៤ សប្តាហ៍)។

### ជំហានទី២៖ BASE (Bielefeld Academic Search Engine)
*   **តំណភ្ជាប់៖** `https://www.base-search.net/`
*   **សកម្មភាពដែលបានធ្វើ៖** បានបំពេញទម្រង់ "Become a content provider"។
*   **ស្ថានភាព៖** បានដាក់ស្នើរួចរាល់ និងកំពុងរង់ចាំការត្រួតពិនិត្យ។

### ជំហានទី៣៖ CORE (Connecting Repositories)
*   **តំណភ្ជាប់៖** `https://core.ac.uk/data-providers`
*   **សកម្មភាពដែលបានធ្វើ៖** បានបង្កើតគណនី CORE រួចយកបណ្ណាល័យទៅចុះឈ្មោះក្នុង Repository Dashboard។
*   **ស្ថានភាព៖** បានដាក់ស្នើរួចរាល់។ CORE នឹងចំណាយពេលរហូតដល់ ៣សប្តាហ៍ ដើម្បីទាញយកទិន្នន័យ ហើយគេនឹងផ្ញើអ៊ីមែលមកប្រាប់នៅពេលរួចរាល់។

### ជំហានទី៤៖ OpenAIRE
*   **តំណភ្ជាប់៖** `https://provide.openaire.eu/`
*   **សកម្មភាពដែលបានធ្វើ៖** ត្រូវបានផ្អាកសិន (Pending)។
*   **លក្ខខណ្ឌ៖** OpenAIRE តម្រូវឱ្យបណ្ណាល័យរបស់អ្នកមានឈ្មោះក្នុងបញ្ជី OpenDOAR (ជំហានទី១) ជាមុនសិន។ នៅពេលដែល OpenDOAR អនុម័តឈ្មោះ PTEC រួចរាល់ ឈ្មោះបណ្ណាល័យនេះនឹងលោតចេញមកក្នុងប្រអប់ជ្រើសរើសប្រទេសរបស់ OpenAIRE នោះអ្នកនឹងអាចបន្តការចុះឈ្មោះនេះរហូតដល់ចប់បាន។
